# Луч × Яндекс Алиса — Квиз-бриф

> Адаптивный квиз для сбора вводных по дизайн-задачам. Next.js 14, TypeScript, SCSS Modules, Vercel KV. Два блока вопросов — Блог Алисы и Дайджест — с механикой ачивок и админкой для просмотра ответов.

---

## О проекте

Инструмент для быстрого брифинга клиента: вместо заполнения документа — короткий квиз с выбором вариантов и свободными текстовыми полями. За каждый заполненный ответ выдаётся ачивка. Собранные данные сохраняются в Vercel KV и доступны в закрытой админке.

Основная задача: снизить порог входа для клиента и получить структурированные вводные для оценки проекта без созвона.

## Архитектура

```
├── components/
│   ├── AchievementToast.tsx   # Тост-уведомление при получении ачивки
│   └── QuestionBlock.tsx      # Рендер вопроса: single, multi, text, multi_with_text
├── lib/
│   ├── questions.ts           # Все вопросы, типы, ачивки
│   ├── storage.ts             # Vercel KV: запись и чтение сабмитов
│   └── uuid.ts                # UUID v4 без внешних зависимостей
├── pages/
│   ├── api/
│   │   ├── submit.ts          # POST /api/submit
│   │   └── submissions.ts     # GET /api/submissions
│   ├── index.tsx              # Квиз (публичный)
│   └── admin.tsx              # Просмотр ответов (закрытый)
└── styles/
    ├── reset.scss             # Глобальный сброс стилей
    ├── globals.scss           # Переменные и токены
    ├── quiz.module.scss       # Стили квиза
    └── admin.module.scss      # Стили админки
```

**Ключевые решения:**

- **Механика ачивок** — каждый вопрос имеет привязанную ачивку с эмодзи и названием. При первом ответе появляется тост-анимация. На финальном экране отображаются все разблокированные ачивки.
- **Типы вопросов** — `single`, `multi`, `text`, `multi_with_text`. Все рендерятся через единый компонент `QuestionBlock` с единой логикой состояния.
- **Vercel KV** — ответы хранятся в Redis: каждый сабмит по ключу `submission:{id}`, список ID в `lpush`-списке. Чтение и запись асинхронные.
- **Без внешних зависимостей для UUID** — генерация через `Math.random()` напрямую, чтобы не тащить лишний пакет.

## Вопросы

### Блок 1 — Блог Алисы (7 вопросов)

Содержимое блога, формат материалов, частота публикаций, целевая аудитория, тон голоса, свободное описание задачи, референсы.

### Блок 2 — Дайджест (6 вопросов)

Формат канала, содержание выпуска, периодичность, целевая аудитория, свободное описание, дизайн-требования + тайминг.

## Локальный запуск

```bash
# Установка зависимостей
npm install

# Настройка окружения
cp .env.example .env.local
# Заполнить переменные из Vercel Dashboard → Storage → KV

# Dev-сервер
npm run dev

# Продакшн-сборка
npm run build
```

---

## Переменные окружения

| Переменная                    | Описание         |
| ----------------------------- | ---------------- |
| `KV_URL`                      | URL Vercel KV    |
| `KV_REST_API_URL`             | REST endpoint KV |
| `KV_REST_API_TOKEN`           | Токен доступа    |
| `KV_REST_API_READ_ONLY_TOKEN` | Read-only токен  |

Все значения берутся из Vercel Dashboard → Storage → твоя KV база → вкладка `.env.local`.

---

## Деплой

```bash
npx vercel
```

После деплоя: Vercel Dashboard → Storage → Create → KV → база автоматически привязывается к проекту и env-переменные проставляются сами.

Просмотр ответов: `/admin`
